Linux入门必会

发表于 2016-06-23   |   分类于 学习总结   |     |   阅读次数

对玩linux的小体会

　　在此，班门弄斧了，个人不是linux大神，但是坚信玩“溜”linux，真的很酷！
对于新手，个人感觉先玩几天虚拟机相对来说，挫败感不会很强，因为如果系统崩了，很容易恢复到以前的状态,而且不会关注很多不必要的东西，曾经装ubantu输入法让我特别恶心。可以按自己的需求使用VMware的快照，以便将系统恢复到自己期望的状态。对于纯linux环境，推荐CentOS，功能全面的发行版，而ubantu也不错，只是增加了图形化界面，相应增加了系统开销。

使用Xshell的第一个坑

　　若用虚拟机，最好还是选择一个了解终端，可以选Xshell，比较简单，这里有个坑。
　　　Xshell链接CentOS, 若ifconfig看不到eth0信息:
　　　　　执行　vi /etc/sysconfig/network-scripts/ifcfg-eth0
　　　　　编辑下里边的BOOT，改为yes后 :wq退出（这是让开机自动eth0）
　　　　　然后service network restart
　　　　　重启网络服务，然后再ifconfig看看有没有eth0

命令基本格式

　　Linux使用命令达三千多个、常用200个、突击使用六七十个。常用命名必须记住。
　　[root@localhost ~]#
　　　　root代表当前登录用户名（管理员账户）非administrator(windows)
　　　　localhost当前计算机主机名，windows在局域网内主机名不能相同
　　　　~ 波浪号代表当前所在目录 此处~为简化符号 代表家 初始根目录 　　　　管理员家为/root 普通用户家为/home/usr
　　命令[选项][格式]
　　注：个别命令使用不遵循此格式
　　当有多个选项时，选项用于调整命令功能，可以写在一起如 ls – ah
　　简化选项与完整选项 –a等于—all

文件处理命令

查询目录中内容
ls(list directory contents)
目录也是文件，目录文件
选项：
　　-a 显示所有文件，包括隐藏文件（linux为 “. ”开头文件，表示系统文件， 非必要不要轻易修改）
　　-l 显示详细内容 ls –l 别名ll
　　如：-rw-r–r–. 1 root root 7690 5月 26 17:43 install.log.syslog
　　　　-文件类型(-文件 d 目录 l软链接文件) 共有7种 块设备文件、字符设备文 件、套接字文件、管道文件
　　　　rw-　　　　　r–　　　　　r–
　　　　u所有者 g所属组 o其他人
　　　　文件权限每三个一组，r读，w写，x执行
　　　　“.”代表”ACL”权限，1表示引用计数 代表这个文件被调用几次，第一个root 表示所有者 第二个root表示所属组 7690表示文件大小 日期表示最后更新日期
　　-d查看目录属性
　　-h人性化显示文件大小
　　-i显示inode　

建立目录

　　mkdir –p 目录名
　　　-p递归创建
　　mkdir –p japan/cangls 创建一串目录必须加-p

切换目录　

cd change directory
　　cd ~ 回到家目录
　　cd
　　cd – 进入上次所在目录
　　cd .. 进入当前目录上一级目录
　　相对路径：参考当前所在目录，进行查找 如[root@localhost ~]# cd ../usr/local/src/ 查找当前目录上一级目录下usr/local/src 目录
　　绝对路径：从根目录开始指定，一级一级递归查找，在任何目录下都能进入指定位置。
　　pwd
　　　　print working directory查询所在目录位置。
　　删除空目录: rmdir
　　rmdir [目录名]
　　　　remove empty directories 只能删除空目录，用得极少
　　rm –rf [文件或目录] remove
　　　　-r 删目录
　　/tmp/
　　加*表示删除tmp下目录和文件 不加表示删除tmp
　　若为-rf false强制删除，不必询问，不能恢复 rm –rf / 自杀，标志root是真正意义上root
　　复制命令:cp
　　cp [选项] [原文件或目录][目标目录或新文件名]
　　　　原英文:copy
　　　　选项: -r 复制目录
　　　　　　　-p 连带文件属性复制
　　　　　　　-d
　　　　　　　-a 相当于-pdr
　　若复制文件且只给出目标目录 则文件名为原文件名，复制后文件的时间属性可能变为当前复制时间 而不是最后修改时间，若要保证相同使用-a
　　剪切或改名命令:mv (move)
操作目录或文件都一样 不加-r
　　若源文件和目标文件在同一个目录下则为改名，不在同一目录则为剪切

常见目录的作用

　　bin boot cgroup dev etc home japan lib lost+found media misc mnt net opt proc root sbin selinux srv sys tmp usr var
　　/ 根目录
　　/bin命令保存目录(普通用户就可以读取的命令)
　　根目录下的bin和sbin，usr目录下的bin和sbin，这四个目录都是用来保存系统命令的。sbin下命令只有root才能执行
　　/boot启动目录，启动相关文件
　　/dev 特殊文件保存目录（硬件文件）
　　/etc 保存系统默认配置文件，在学习一些服务时会用到
　　/home 普通用户家目录 /root超级用户家目录
　　/lib linux中函数库保存目录，将某些功能封装成函数库，在需要的时候才调用
　　/media /mnt /misc三个空目录，所有的存储设备都要挂载后使用，挂载可以当做分配盘符，只要是空目录就行，用这三个空目录来最为外接设备盘符，misc用来挂外接磁带机，media用来挂光盘，mnt挂载u盘或移动硬盘
　　/proc sys目录不能直接操作，这两个目录保存的是内存的挂载点。这两个目录中的数据是直接写在内存中的。
　　/tmp临时目录，存放临时数据
　　/usr系统软件资源目录
　　/usr/bin/系统命令(普通用户)
　　/usr/sbin/系统命令（root用户）
　　/var保存系统可变文档目录
　　可以在家目录root或home,以及tmp目录下随便放内容，做练习，不推荐在根下做练习

挂载命令

w或者who查看系统中登录了哪些用户，即查询系统中出了当前用户外，还有多少其他用户。只是w能看到一些系统资源的使用情况。

tty1直接登录用户1
pts/0 远程登录用户1
-bash等待输入命令
last命令
查询当前登录和过去登录的用户信息，即判断系统中所有用户登录信息
last命令默认是读取/var/log/wtmp文件数据,该文件为二进制文件。必须用last而非vi查看
lastlog查看所有用户的最后一次登录时间

echo命令的功能是在显示器上显示一段文字，一般起到一个提示的作用。在shell编程中极为常用, 在终端下打印变量value的时候也是常常用到的
echo [ -n ] 字符串
   选项n表示输出文字后不换行；字符串能加引号，也能不加引号。
   用echo命令输出加引号的字符串时，将字符串原样输出。
   用echo命令输出不加引号的字符串时，将字符串中的各个单词作为字符    串输出，各字符串之间用一个空格分割。
   若字符串中有特殊的如\t \a \b \n等转义字符时，按照转义输出。

alias 查看系统中所有命令的别名
alias别名=’原命令 设定别名 重启失效

vi ~/.bashrc 写入环境变量配置文件 永久生效
unalias 别名 删除别名

除非你确定要覆盖原有命令，否则新建的别名不能和原有的命令重名。

history [选项][历史命令保存文件]
选项:-c 清空历史命令
-w把缓存中的历史命令写入历史命令保存文件~/.bash_history

wc [选项][文件名]
-c统计字节数
-w统计单词数
-l统计行数
用于软件安装，打补丁
以文件作为输入源，而非键盘
wc << 字符串 统计两相同该字符串间字符情况，实际中此命令用得很少

命令1|命令2
若命令1正确执行，将命令1的执行结果传递给命令2，要求命令2必须能操作命令1的结果
如 ls –l /etc | more 将命令1的结果分屏显示
netstat –an | grep ESTABLISHED 在命令1的结果中查询包含ESTABLISHED的行，查看多少人连接了服务器。
netstat –an | grep ESTABLISHED |wc –l 直接连接统计个数
